# OctoPrint-E3S1P_ByTT Thumbnails

Enhanced thumbnail support for Ender 3 S1 Pro firmware fork, with a modern, reliable architecture designed for real-world OctoPrint workflows.

---

## 🚀 Overview

This plugin builds upon the original thumbnail extraction work by **[jneilliii](https://github.com/jneilliii/)**  and subsequent enhancements by **[Thomas Toka](https://github.com/ThomasToka/)**, extending it with a fully refactored, event-driven design.

It enables reliable thumbnail handling across all common usage patterns, including slicer direct-print workflows and multiple uploaded files.

---

## ✅ Supported Workflows

This plugin has been tested and works reliably with:

* Upload → print later from OctoPrint
* Direct print from slicers (OrcaSlicer, PrusaSlicer, etc.)
* Upload without printing (multiple files queued)
* Printing files in any order
* Printing after OctoPrint restart
* Canceling a print and starting another

---

## 🧠 How it works

1. A G-code file is uploaded to OctoPrint
2. The plugin extracts the embedded thumbnail
3. The thumbnail is stored and associated with that file
4. When a print starts, the correct thumbnail is sent to the printer

This event-driven approach ensures consistent behavior regardless of how the print is initiated.

## ⚙️ Requirements

* OctoPrint 1.8+ (recommended)
* Firmware that supports thumbnail display (e.g. Thomas Toka [E3S1PROFORKBYTT firmware fork](https://github.com/ThomasToka/MarlinFirmware))
* Post-Processing Script installed in slicer

---

## ⚡ Quick Setup

1. Install the plugin in OctoPrint  
2. Configure your slicer to use the correct post-processing script (see below)  
3. Slice and upload a model  
4. Start the print — the thumbnail should appear on your printer  

If thumbnails do not appear, double-check your post-processing script configuration.

---

## ⚠️ Required: Post-Processing Scripts

This plugin **will not function correctly without a post-processing script**.
If thumbnails are not properly embedded and formatted, the printer will not display them even though the plugin is installed.

This plugin relies on data generated by a post-processing script executed by the slicer. Depending on the slicer, these scripts convert PNG thumbnails to JPG, normalize the embedded thumbnail block, and inject print metadata required by the firmware.

Choose the script that matches the slicer and version you are using. The links below point to the original upstream scripts in Thomas Toka's [MarlinFirmware](https://github.com/ThomasToka/MarlinFirmware) repository:
